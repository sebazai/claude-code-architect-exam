# Claude Certified Architect – Foundations Practice Exam

An interactive practice exam for the **Claude Certified Architect – Foundations** certification, running entirely inside Claude Code via MCP. No `ANTHROPIC_API_KEY` required in the server process — the **exam** MCP server calls the **Claude Code CLI** (`claude -p`) for each generation and quality-evaluation pass. The separate **evals** MCP server (prompt benchmarking) uses **MCP Sampling** (`ctx.sample()`).

## What This Is

A study tool that generates unique, scenario-grounded exam questions on demand. Each question is generated by Claude, quality-evaluated by a second Claude pass, and retried with feedback if it falls below the quality bar — all before you see it.

- **Full exam** (`start_exam`): **60 questions** across 4 randomly-selected scenarios (15 per scenario), **120-minute** user-active timer
- **Mini exam** (`start_exam_mini`): **20 questions** (4 scenarios × 5 questions), **40-minute** user-active timer — same flow and scoring model; time is scaled linearly from the full exam
- Generation and evaluation time is excluded from the timer in both modes
- **Score 100–1,000** (passing: ≥ 720), with a per-domain breakdown at the end

## Scenarios

Each session randomly selects **4 scenarios** from a **pool of 13**. The official exam guide lists 6; this repo includes those plus 7 additional scenarios aligned with the broader real-exam pool. See `exam-app/mcp_server/exam_content.py` (`SCENARIOS`) for names and descriptions.

The six scenarios from the official guide are:

1. Customer Support Resolution Agent
2. Code Generation with Claude Code
3. Multi-Agent Research System
4. Developer Productivity with Claude
5. Claude Code for Continuous Integration
6. Structured Data Extraction

## Running the Exam

### Option 1 — Dev Container (recommended)

Fully isolated, no local Python or Node.js setup needed.

1. Clone the repo and open it in VS Code.
2. When prompted, click **"Reopen in Container"** (or run `Dev Containers: Reopen in Container` from the command palette).
3. The container installs all dependencies automatically on first start.
4. Open the Claude Code chat and type:

```
Start the Claude Certified Architect exam
```

The devcontainer bind-mounts `exam-app/.mcp.json` to `/workspace/.mcp.json`, so Claude Code picks up the MCP server automatically.

### Option 2 — Local

Install both packages so every server in `.mcp.json` can start (the file registers the exam server and the evals server):

```bash
pip install -e exam-app/ -e evals/
```

Open Claude Code from the **repository root** with `exam-app/.mcp.json` symlinked or copied to `.mcp.json`, **or** from `exam-app/` if that directory’s `.mcp.json` is the one Claude Code loads. Then:

```
Start the Claude Certified Architect exam
```

## How It Works

```
Claude Code (host / MCP client)
  │
  ├── start_exam          → full exam: 4 random scenarios, 60 Q, 120 min active time
  ├── start_exam_mini     → mini: same 4-scenario flow, 20 Q (5 per scenario), 40 min active time
  ├── get_next_question   → generate → quality-eval → retry if score < 3/5
  ├── submit_answer       → record answer + user think-time, return verdict
  ├── exam_status         → check progress without advancing
  └── get_results         → scaled score, pass/fail, domain breakdown
```

Question generation is an agentic loop:

1. Build a prompt from the scenario description, domain concepts, anti-patterns, and 2 few-shot examples.
2. Call the Claude CLI to generate a question.
3. Run a second CLI pass to score quality (1–5).
4. If score < 3, inject the evaluator's feedback into the prompt and retry (up to 2 retries).
5. Accept the best result after retries are exhausted.

The user-active timer pauses during steps 2–4. You are only timed while reading the question and choosing your answer.

## Reviewing Session Activity

The MCP server exposes a session log as a resource. To inspect generation and quality-eval events mid-exam:

```
Read the resource exam://session/log
```

This shows each question's generation attempt, quality score, and any retry triggers.

## Scoring

The scaled score is always **100–1,000** with the same passing threshold (**≥ 720**), whether you use the full exam or the mini exam. The formula is linear in your accuracy: `correct ÷ total_questions` maps to the scale (0 correct → 100, all correct → 1000).

| Raw correct (full exam, of 60) | Scaled score |
|---|---|
| 60 | 1000 |
| ~39 | 720 (passing) |
| 0 | 100 |

For **mini** (`start_exam_mini`, 20 questions), the same interpolation applies with **20** as the denominator (e.g. all 20 correct → 1000). The `get_results` tool returns your total score, pass/fail status, and accuracy per domain so you can identify weak areas.

## Eval Pipeline

The `evals/` directory contains a prompt evaluation pipeline for the exam simulator's generation and quality-evaluation prompts.

### Two ways to run

**Via MCP (no API key needed)** — the evals MCP server uses `ctx.sample()` (not the same mechanism as the exam server, which uses the `claude` CLI):

```
Run the generation eval for variant v2 with 5 cases
Run the calibration eval
Run the full eval comparing v1, v2, and v3
```

The `claude-architect-evals` MCP server is registered in `.mcp.json` alongside the exam server.

**Via CLI (requires `ANTHROPIC_API_KEY`)**:

```bash
cd evals
python run_evals.py --eval-type calibration --skip-meta   # quick calibration only
python run_evals.py --variants v1,v2 --n-gen 5            # compare two variants
python run_evals.py --eval-type all                        # full suite, all variants
```

### What it measures

**Generation eval** — for each of 3 prompt variants, generates questions and grades them:
- Code grader: structural validity (12 checks — JSON shape, distinct options, explanation length, no "all of the above", etc.)
- Keyword grounding: does the question reference scenario-specific terms?
- Quality eval: the existing 1–5 rubric via sampling
- Meta-grader: independent 5-dimension rubric (scenario grounding, tradeoff reasoning, distractor quality, explanation completeness, single correct answer)

**Calibration eval** — tests whether the quality evaluator correctly scores 15 pre-labeled questions (5 good / 5 mediocre / 5 bad). Reports accuracy by label.

### Prompt variants

| Key | Name | Change |
|-----|------|--------|
| `v1` | Baseline | Current production prompt (unchanged) |
| `v2` | Anti-Generic | Adds explicit constraint requiring scenario-specific grounding |
| `v3` | Chain-of-Thought | Adds `<thinking>` step before JSON generation |

## Study Resources

The official exam guide (`claude_certified_architect_instructions.pdf`) covers all domains in detail. Key decision-making themes the exam tests:

- Programmatic enforcement over prompt-based guidance for critical business logic
- Explicit criteria over vague instructions
- Structured error context in multi-agent systems
- Tool descriptions as the primary mechanism for tool selection
- Multi-pass reviews to avoid attention dilution
- Structured fact extraction over growing raw conversation context
